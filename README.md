# Text-Conditioned Rectified Flow Matching in Pixel Space

This repository implements a text-conditioned image generation model trained
using Rectified Flow Matching. The model operates directly on pixel data:
images are generated by numerically integrating a learned velocity field
starting from Gaussian noise, without an intermediate compressed
representation. All parameters are randomly initialized prior to training;
no pretrained components are used. Training was performed on two NVIDIA T4
GPUs (Kaggle) using a dataset of approximately 21,551 anime-style face
images paired with text embeddings.

## Overview

The model is a UNet-based architecture conditioned on text through
cross-attention layers, trained with a rectified flow matching objective to
predict a constant velocity field between a noise distribution and a target
image distribution. Because the model operates on raw pixel data, rather
than a compressed representation, architectural decisions such as attention
placement and weight initialization have a direct and immediately
observable effect on sample quality.

The model generates 64x64 pixel anime-style face portraits conditioned on
free-form text descriptions. Text conditioning is provided through
512-dimensional embeddings computed from a CLIP text encoder.

## Architecture

The model uses an ADM-style UNet backbone (Dhariwal & Nichol, 2021) with
Imagen-style text cross-attention (Saharia et al., 2022):

```
Input (3, 64, 64)
  -> Input convolution
  -> Encoder: 3 resolution levels (64 -> 32 -> 16), 2 residual blocks per level
       Self-attention and cross-attention applied only at 32x32 and 16x16
  -> Middle block: residual block -> self-attention -> cross-attention -> residual block
  -> Decoder: mirrors the encoder, with skip connections from the encoder
  -> Output convolution (zero-initialized)
Output: predicted velocity field, shape (3, 64, 64)
```

| Component | Value |
|---|---|
| Base channels | 128 |
| Channel multipliers | 1, 2, 4 |
| Residual blocks per resolution level | 2 |
| Attention resolutions | 32x32, 16x16 |
| Attention head dimension | 64 |
| Text embedding dimension | 512 |
| Context tokens per embedding | 16 |
| Timestep conditioning | Scale-shift normalization (AdaGN) |
| Total parameters | approximately 72.7 million |

### Attention placement

Self-attention operates on a flattened sequence of spatial tokens and does
not, by itself, encode positional information: a token from one spatial
location is indistinguishable from a token at another location unless
positional information is supplied through some other mechanism. Applying
self-attention at the model's full input resolution (64x64), in combination
with horizontal-flip data augmentation, resulted in generated images
exhibiting a vertical discontinuity along the image midline, as the two
halves of each face were generated without positional correlation between
them.

Convolutional layers implicitly encode positional information through
zero-padding at the boundaries of each feature map, which is sufficient to
maintain spatial coherence at full resolution without attention layers.
Attention is therefore applied only at the 32x32 and 16x16 resolution
levels. This follows the convention used in GLIDE (Nichol et al., 2021),
Imagen, and ADM.

## Training Methodology

**Objective.** The model is trained using Rectified Flow Matching (Liu et
al., 2022). Given a data sample x1 and a Gaussian noise sample x0, the
interpolated point along a straight-line path is defined as
zt = (1-t)x0 + t*x1 for t in [0, 1]. The model is trained to predict the
constant velocity v = x1 - x0 at each interpolated point, using a mean
squared error loss.

**Timestep sampling.** The timestep t is sampled as t = sigmoid(u), where
u ~ N(0, 1), rather than uniformly on [0, 1]. This concentrates training on
intermediate timesteps, where the denoising task is most ambiguous,
following the timestep sampling scheme used in Stable Diffusion 3 and Flux.

**Sampling procedure.** Image generation uses deterministic Euler
integration of the learned ODE over 30 to 50 steps. Classifier-free guidance
is enabled by replacing the text embedding with a null (zero) embedding for
15% of training batches, allowing guided sampling at inference time.

**Optimization.** Parameters are optimized using AdamW, divided into three
parameter groups with distinct learning rates: backbone parameters (1e-4),
self-attention parameters (5e-4), and cross-attention parameters (8e-4). A
500-step linear warmup is followed by cosine learning rate decay. The
elevated learning rate for cross-attention parameters compensates for the
comparatively slower convergence of cross-modal attention, in which query
and key representations originate from two initially independent
distributions (image features and text embeddings), as opposed to
self-attention, in which query and key representations are both derived
from the same feature map.

**Hardware and dataset.** Training was performed on two NVIDIA T4 GPUs
(14.5 GB each) using mixed-precision (FP16) training via the Hugging Face
Accelerate library. The training set consisted of 21,551 images at 64x64
resolution, trained for 100 epochs.

## Implementation Details

**Zero-initialization of attention output projections.** Standard practice
zero-initializes a network's final output layer so that training begins
from a stable, low-variance prediction. Zero-initializing the output
projection of each attention block in addition to the network's final
output layer is unnecessary and was found to introduce a secondary
near-zero gradient bottleneck specific to the attention pathway, delaying
the point at which cross-attention parameters received a meaningful
training signal relative to the convolutional backbone. This was identified
through controlled comparison of training dynamics under both
initialization schemes, and resolved by initializing attention output
projections with a small non-zero standard deviation instead.

**Interaction between attention placement and data augmentation.** Random
horizontal flip augmentation, applied in combination with full-resolution
self-attention, produced a directional inconsistency: a given hairstyle and
its mirrored counterpart were both presented as valid training targets for
visually similar inputs, with no positional mechanism available to the
attention layer to reconcile the two. This interaction contributed to the
vertical discontinuity artifact described above. The issue was resolved by
removing the flip augmentation and restricting attention to lower
resolution levels, as described in the Architecture section.

**Evaluation methodology.** Comparing multiple generated samples to one
another does not, by itself, establish whether text conditioning has a
causal effect on the output, since independently generated samples differ
in both their text embedding and their random initialization. To isolate
the effect of conditioning, an evaluation procedure was used in which two
different text embeddings are decoded from an identical starting noise
sample, holding all other variables fixed. This procedure was run
automatically during training and logged alongside the standard training
diagnostics, and was used to confirm that conditioning strength increased
over the course of training.

**Gradient anomaly detection.** In addition to a fixed absolute threshold
for detecting gradient explosion, the training loop compares each epoch's
gradient norm against a trailing median of recent epochs from the same run.
This detects large relative deviations that remain below an absolute
catastrophic threshold but are inconsistent with a given run's own
observed behavior.

## Limitations

- Color attributes (hair color, eye color, and combinations thereof)
  generalize reliably to text prompts not present in the training set.
  Descriptions of hairstyle geometry, such as twin-tail hairstyles, do not
  reliably produce the corresponding structure. This is likely attributable
  to the distribution of the automatically generated training captions,
  which are presumed to describe color attributes more frequently and more
  consistently than structural attributes, rather than to a defect in the
  training procedure.
- Interpolating between two text embeddings does not necessarily transition
  all encoded attributes at a uniform rate. This reflects the geometry of
  the underlying text embedding space, in which distinct semantic
  attributes are not represented as independent, evenly spaced directions.
- Resolution is fixed at 64x64 pixels, which constrains the level of fine
  detail achievable in generated images. This was a deliberate choice given
  the available computational budget.
- No quantitative evaluation metric, such as FID, was computed against a
  held-out reference set. Evaluation relies on the conditioning probe
  described above and manual inspection of samples generated from held-out
  prompts.

## Usage

### Training

```
RFM_DATA_DIR=/path/to/dataset python master_pipeline.py
```

If `RFM_DATA_DIR` is not set, the script defaults to the Kaggle input path
used during original training.

The dataset directory must contain paired files: an image file (`<name>.png`
or `<name>.jpg`) and a corresponding embedding file (`<name>.pt`), where
each `.pt` file is a serialized `torch.Tensor` of shape `[512]` representing
that image's text embedding.

At the end of each epoch, the training loop writes a model checkpoint, an
exponential moving average (EMA) checkpoint, a validation image grid
generated from four fixed text embeddings, a conditioning probe image (two
samples generated from identical starting noise and two different text
embeddings), and a JSON file containing training diagnostics.

To resume training from an existing checkpoint, set `RESUME = True` in the
`run_training()` function.

### Inference

```
RFM_CKPT_PATH=/path/to/checkpoint.pt RFM_DATA_DIR=/path/to/dataset python inference.py
```

This script generates images from a set of example text prompts defined in
the `enhanced_prompts` list, along with a sequence of images produced by
linearly interpolating between two text embeddings while holding the
starting noise fixed. Output images are saved as PNG files in the working
directory.
